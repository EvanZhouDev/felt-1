# Felt-1

**Felt-1 is an isoneural converter: it regenerates an artifact in a different
medium so that, according to a brain-encoding model, both evoke the same
neural response.** A song becomes a paragraph that *feels* like the song. A
painting becomes a different image with the same charge. Not summarization,
not style transfer in pixel space — the invariant is predicted brain
activation.

| Target (Van Gogh, *The Starry Night*) | Generated by Felt-1 (never saw the title — only the pixels) |
|---|---|
| ![target](docs/images/demo-starry-target.jpg) | ![generated](docs/images/demo-starry-generated.png) |

The generated image matches the target at **0.79 neural similarity under our
anchored metric** (0.82 raw) after a five-round search with two candidates
per round. The generating agent was given pixels with an anonymized filename
— it wrote a four-vortex composition prompt from looking, and a search loop
climbed in prompt space. (An earlier internal score of 0.97 turned out to be
inflated by the target leaking into the metric's anchor baseline; we fixed
the methodology and report the clean number. The audit trail is in the
repo.)

## How it works

The shared "vibe space" is Meta's **TRIBE v2** — a model trained to predict
human fMRI responses to video, audio, and language. Run anything through
TRIBE and you get a predicted brain-activation trajectory over 20,484
cortical vertices. Two artifacts in *different* media become comparable in
*one* space.

TRIBE stays frozen. Felt-1 **searches** over outputs:

```
target (song/image/text)  ──render──►  TRIBE  ──►  target activation

   ┌──────────────── one round (repeats) ────────────────┐
   │  ranked past attempts + judge critique               │
   │        ──►  N agent candidates                        │
   │        ──render──►  TRIBE  ──►  score vs. target       │
   │        ──►  re-rank, judge critiques the new best      │
   └──────────────────────────────────────────────────────┘

           ──►  best output whose vibe matches the target
```

The search is **Ranked-Reflect** — an [OPRO](https://arxiv.org/abs/2309.03409)
"LLM as optimizer" loop with [Reflexion](https://arxiv.org/abs/2303.11366)
verbal feedback. Each round, candidate agents see the *ranked* history of past
attempts plus a one-line critique of the current best, and try to beat it.
There are no hand-coded mutation operators. Two anti-reward-hacking guards
feed each candidate's score: text-trigram novelty (no resubmitting the
leader) and **activation-space novelty** (a new wording of the same neural
point is not progress — the trajectory even tells agents when their attempts
have collapsed into one attractor, and demands divergence in *form*).

Image outputs are generated, not retrieved: agents emit `flux:<prompt>`
pseudo-URIs, the orchestrator materializes them through a hosted Flux model,
and the optimization climbs in prompt space. Audio inputs get a tiered
describer (hosted Qwen2.5-Omni caption + a local DSP pass for tempo, energy,
brightness, key) so agents can "hear" what they're matching. Agent backends
are pluggable with automatic failover (`VOLTA_AGENT_BACKEND="codex,claude"`)
— a usage cap on one CLI fails over instead of killing the run.

## Measuring "feel" honestly

The hardest part wasn't generation — it was the metric. Raw cosine over TRIBE
activations is dominated by a *modality common mode*: "this is music being
heard" / "this is English being read." Under raw scoring, three emotionally
opposite pieces (Clair de Lune, Moonlight Sonata 3rd mvt, Dvořák's New World
finale) are **0.88–0.92 similar to each other**, and a hand-written *calm*
text outscores a hand-written *storm* text **on the storm piece**. Any search
under that reward converges to generically-evocative mush — ours did.

The fix: **anchor subtraction**. We encode a small diverse corpus per
modality (neutral texts, varied audio *including synthesized speech*, neutral
generated images), and subtract the corpus mean from every activation frame
before scoring. After anchoring, wrong-emotion pairs go *negative* (the calm
text scores −0.42 against Moonlight) and matching texts win their targets
with real margins. For cross-modal audio→text we add a contrastive term
against the anchor battery: *match this piece beyond matching audio in
general*.

Specificity is verified with cross-score matrices: every winning output is
re-encoded and scored against **every** target. The output for piece A must
beat the outputs for pieces B and C *on A's own column*.

What the matrices say today (real hosted TRIBE, full runs):

- **Image→image: 0.79 anchored similarity** (0.82 raw) on a tiny search
  budget, and image→text is target-specific out of the box (2/2 diagonal
  with real margins).
- **Audio→text: 2/3 diagonal** after the metric work (was 1/3 with the
  attractor winning every column). The residual confound is mechanical:
  TRIBE renders text via speech synthesis, so every text is acoustically
  *speech* — quiet piano resembles speech more than a fortissimo orchestra
  does. Widening the audio anchor battery measurably shrinks it; it is a
  corpus-coverage problem, not a wall.
- The search loop improves across rounds (e.g. 0.700 → 0.779 over five
  rounds on Moonlight) instead of plateauing at round 1, which is where it
  started the week.

All of this is reproducible: `experiment-matrix.ts` runs target sets end to
end and emits the matrices; `metric-probe.ts` / `metric-text-probe.ts` /
`metric-contrast.ts` are the metric validation harness; `build-anchors.ts`
rebuilds the anchor corpus (per-stimulus cache, anchors ship in the repo).

## Layout

Bun monorepo:

- `packages/core` — contracts (nodes, payloads, activation traces) and the
  scoring algorithm (`src/scoring/activation.ts`: anchored blend of pooled,
  temporal, dynamics, and best-match cosines).
- `packages/agent-sdk` — candidate/judge prompts and the agent backends
  (Codex CLI, Claude Code CLI, failover chain).
- `services/orchestrator` — the search loop, TRIBE oracle (mock / local
  Python / hosted HTTP), Flux image generation, audio describer, anchors,
  experiment harness, SQLite run index, readable per-run JSON artifacts.
- `apps/web` — Next.js trace explorer.
- `vendor/tribev2` — vendored Meta TRIBE v2 (frozen oracle; do not edit
  beyond the documented Mac-MPS patches).

## Quick start

```bash
bun install
bun run check          # biome lint + typecheck

# Fast, offline, mock oracle — proves the wiring end-to-end:
bun run smoke          # text → text
bun run smoke:audio    # audio → text
bun run smoke:image    # image → text

# Real runs (hosted TRIBE oracle; agent CLI required — codex or claude):
VOLTA_ORACLE=http VOLTA_SMOKE_IMAGE=path/to/photo.jpg bun run smoke:image
VOLTA_ORACLE=http VOLTA_DESCRIBE_AUDIO=true \
  VOLTA_SMOKE_AUDIO=path/to/song.mp3 VOLTA_SMOKE_OUTPUT=image bun run smoke:audio

# Reproduce the cross-score experiments:
VOLTA_AGENT_BACKEND=claude bun run exp:matrix my-exp path/to/targets.json
```

Each run writes inspectable artifacts (target, per-iteration trajectory,
scores, judge critique, score curve) under the printed run root. Config knobs
are documented in [CLAUDE.md](CLAUDE.md).

## Honest limitations

- Cross-modal absolute similarity is bounded by TRIBE's rendering paths
  (text→speech, image→still video). Same-modality transfer reaches ~0.97;
  cross-modal lives lower and specificity is the meaningful claim.
- The anchor corpus is small (8 texts / 10 audio / 7 images). Specificity
  improves monotonically with corpus diversity in our ablations; it is the
  highest-leverage place to contribute.
- N=2 candidates × 4–5 rounds per run is a hackathon budget. Score curves
  suggest wider rounds would help more than deeper ones.

TRIBE weights stay frozen. Felt-1 owns the agentic search layer around it.
