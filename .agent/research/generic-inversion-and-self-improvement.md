# Volta: Generic Any-Medium Inversion & Self-Improving Orchestration

## 1. Is generic any-medium inversion possible?

**Honest answer: Not as a single generic move, but yes per-medium under one hard condition — you must supply a generator (or a sampleable renderer) for that medium.** The "feed a 20484-dim TRIBE activation into a universal conditional generator" dream is dead on arrival for two confirmed reasons in our setup:

- **No gradients, non-differentiable render boundary.** `tribe_oracle_worker.py` is stdin/stdout JSON returning a `values` tensor — zero autograd surface. And the render path (text→events, image→mp4, code→screenshot→video) is non-differentiable. This **categorically rules out** every inference-time-guidance method (CLIP-guided generation, FuseDream, ReNO, ImageBind/Seeing-and-Hearing aligners). Anyone who proposes "backprop the alignment score" is wrong for Volta. Don't chase it.
- **TRIBE-brain-space is alien to every off-the-shelf generator's conditioning space.** CoDi / Unified-IO 2 condition on *their* latent, not a cortical vector. There is no slot to inject ours.

**What survives, and the conditions under which it works:**

The only framing that generalizes is the **inverse-problem framing**: TRIBE is a brain *encoder*, so "render an output matching activation A" is literally fMRI-to-stimulus decoding. Two regimes work:

- **Search (works TODAY, zero training):** optimize a *low-dimensional latent* of a frozen per-medium generator with a gradient-free optimizer (CMA-ES / TuRBO / Bayesian opt), scored by the scalar TRIBE-similarity Volta already computes. This is Volta's current loop with a continuous knob instead of an LLM proposer. No gradients, no paired data, no cross-modal reasoner. Proven on GANs and one-step diffusion with non-differentiable rewards (ZeNO; surrogate-latent BBO arXiv:2509.23800; Tripp et al. weighted retraining, ~500 evals).
- **Amortize (research-grade, one-time cost per medium):** manufacture an `(output, activation)` dataset by self-querying frozen TRIBE, then train a conditional generator `activation→medium` (ID3PM-style classifier-free-guidance diffusion inverts a *black-box* recognizer with forward embeddings only; MindEye2/Takagi-Nishimoto do this for images). Inference is one fast forward pass, oracle out of the loop.

**Where it breaks — be honest about the three walls:**

1. **The query-budget × dimensionality wall is the binding constraint.** The near-exact published analog (Black-Box Adversarial Prompting, arXiv:2302.04237) spends **5,000–10,000 queries to tune 4–6 tokens**. At our **600s/query, concurrency=1** oracle, blind search over raw tokens/pixels is infeasible. The latent generator is what collapses dimensionality enough to make search converge (CMA-ES degrades past ~100 dims; vanilla BO past ~15–60). **No generator for the medium ⇒ no tractable search.** This is the real meaning of "generic": it's only as generic as your supply of per-medium generators.
2. **Inversion is ill-posed (many-to-one).** Many outputs share a vibe; a plain regression inverse collapses to the average. You need a generative prior (tandem-net / cGAN / CFG) or you get adversarial "fooling" outputs.
3. **The reward metric is gameable, and that poisons everything above.** Our own memo documents full-vector cosine reward-hacking (repetition + generic-language baseline). The scorer `packages/core/src/scoring/activation.ts` already moved to a blend (pooled 0.4 / trajectory 0.3 / best-match 0.3) which is *better* but any aggressive optimizer — search or learned — will hunt the metric's residual exploits. **Harden the metric before you scale the optimizer.**

## 2. Minimal architecture to add a NEW output medium generically

Two-tier. Ship Tier 1; reach for Tier 2 only when per-target query cost forces amortization.

**Tier 1 — NOW, zero training (drop-in to the existing loop):**
```
{ frozen deterministic generator for medium X (DDIM / flow / GAN — latent fully specifies output) }
  + { render(X) → TRIBE-query wrapper exposing one scalar = scoreActivations vs target }   ← already exists
  + { gradient-free optimizer over a LOW-dim latent (CMA-ES via `cma`, or TuRBO) }
```
That is the entire stack. The non-differentiable render boundary is a non-issue because the optimizer never differentiates. To add medium X you supply exactly one thing: a generator with a sampleable latent. Budget hundreds of oracle calls per target; keep the latent ≤~100 dims or trust-region-restricted; keep candidates on-manifold (the generator's prior does this for free) and add a held-out check to resist reward-hacking.

**Tier 2 — amortized, one-time per medium (when slow oracle is the bottleneck):**
```
1. sample 10k–100k outputs in medium X (from a base generator / corpus)
2. render + query frozen TRIBE → label each with its activation  ← the only expensive, one-time step
3. train a small CFG conditional generator  g(target_activation) → X   (ID3PM recipe)
4. inference = single forward pass, optionally + a few CMA-ES latent-refinement steps
```
The MindEye2/unCLIP shortcut (`activation→CLIP→Flux`) is *only* available for **image** output — it borrows another medium's already-solved inverse and a frozen shared embedding. It **fails by construction for a truly novel medium** with no foundation embedding. Don't sell it as generic; sell it as "the cheap image path we already have a generator for."

**Ranked by feasibility-for-Volta:** (1) **CMA-ES/TuRBO over a frozen generator latent** — only thing that works today, medium-agnostic, no training. (2) **ID3PM-style amortized decoder** — the genuinely generic long-term answer, expensive once. (3) **Surrogate-of-TRIBE** — recovers a gradient and slashes oracle calls, but discrete-media render breaks the differentiable path and surrogate error invites reward-hacking. (4) **activation→CLIP→generator adapter** — image-only. (5) **inference-time guidance / shared aligners** — ruled out, gradients unavailable.

## 3. Single most promising path to a self-improving orchestrator

**Volta is already past the starting line — do not rebuild it.** `run.ts` (`planOperators`, lines 965–1031) is a real UCB1 adaptive-operator-selection bandit over a 12-operator library, fed by `operatorFitnessStats(archive)`, with stall-driven exploration widening (`explorationWeight` 1.5 vs 0.6, line 993) and elitism. `archive.ts` is a FunSearch-style program database (`archivePromptContext` top/diverse/recent + `bestPerBehavior` MAP-Elites coverage). The gap the question targets — "smarter *over runs*, which operator for which *target type*" — is small and cheap to close.

**Single best path: make the bandit CROSS-RUN and TARGET-CONDITIONED (a contextual bandit at "target type" granularity).** Concrete steps in our code:

1. **Fix the credit signal.** In `planOperators` / the credit-assignment feeding `operatorFitnessStats`, change operator reward from **absolute `neuralSimilarity`** to **fitness IMPROVEMENT over the parent elite**. The current absolute-score credit biases toward late-iteration operators that *inherit* a high parent score. Improvement-over-parent is the AOS literature's recommended signal (Fialho/DaCosta) and is the highest-leverage one-line-of-intent change.
2. **Persist a global `(operator × targetBucket)` reward table.** `targetBucket` is a cheap, medium-agnostic key already computable from `target.json` diagnostics — **dominant Yeo-7 network** (you already have `probe-yeo.ts` and `diagnostics.yeo7Means`), falling back to input medium. Have `appendTargetCandidateArchive` (archive.ts:58) also write to this global table, not just the per-`targetSha` archive.
3. **Warm-start `planOperators` priors** from that global table for the matching bucket instead of the flat optimistic prior — and flip `VOLTA_REUSE_TARGET_ARCHIVE` on for *matching buckets* (today it defaults false and only reuses the *identical* `targetSha`, archive.ts:49–67).

This is exactly the asked-for behavior ("which operators worked for which target type"), costs **almost zero extra oracle queries**, needs no gradients / no generative model / no cross-modal reasoner, and is medium-agnostic because both the operators and the Yeo descriptor are.

**Layer second (cheap, high leverage): a Reflexion-style offline meta-reflector.** After each run, a *separate* meta-LLM (never touches the medium — only the logs) reads `evolution-journal.json` / `scores.json` / `judge.json` and appends natural-language lessons ("novelty-injection only paid off after stalls; inventory-syntax operator collapsed scores on high-energy Yeo-visual targets") to a persistent strategy memory injected into the next run's planner prompt. You already write exactly the artifacts a Reflexion actor needs. It reasons over scores + operator labels only, so it survives a medium with no reasoner, and can eventually rewrite the operator instruction strings (PromptBreeder).

## 4. What is genuinely hard or open

- **The metric is the reward, and it's exploitable — this gates everything.** Self-improvement *amplifies* reward-hacking: a smarter controller optimizes the cosine-blend's residual exploits faster. **Hardening the metric must precede aggressive meta-learning.** This is the single most important open problem; the operator-evolution and surrogate paths are unsafe until it's addressed.
- **Slow oracle starves every data-hungry method.** At 600s/query, concurrency=1, the meta-dataset for transfer-BO / amortized inversion / surrogate training accrues painfully slowly. This is why full **AlphaEvolve/PromptBreeder self-rewriting of the operator library** and **meta-learned warm-starting of M/N/diversity hyperparameters** are *deferred*, not chosen — they need dozens-to-hundreds of past runs or a held-out target benchmark to beat the cheap bandit-table, and we can't accrue that yet.
- **Is TRIBE invertible at all?** It pools to brain-region responses and may be genuinely many-to-one (the question warns "may lose vibe, keep only topic"). An amortized decoder can only recover what the activation actually carries. **Unvalidated** — this is what exp-1/exp-2 in MEMORY should pin down before betting on amortization.
- **The generator prerequisite is the true boundary of "generic."** For a medium with *no* generator and *no* foundation embedding — precisely the case the brief flags — there is no tractable path. Honest position: Volta is generic over *media that have a generator*, plus its genetic-operator loop as the universal (but least sample-efficient, rugged-landscape-degrading) fallback when all you can define is one valid mutation.

**Bottom line:** Generic inversion is *partially tractable* — feasible per-medium when you supply a generative prior plus a surrogate to cut oracle calls; **not** feasible as a single generic blind search. The self-improving orchestrator is *tractable now* at the contextual-bandit + reflection layer (small, cheap edits to `run.ts`/`archive.ts`), and *research-grade* for operator-library self-evolution — blocked first on hardening the score metric.

## Per-subquestion detail

### blackbox-search: Gradient-free / black-box optimization against an expensive frozen oracle (TRIBE) to find a renderable output whose activation matches a target. How well do CMA-ES, NES, Bayesian optimization, simulated annealing, and particle methods scale to a discrete/structured search space (text tokens, rendered media) rather than a fixed-dim continuous vector, and what makes the oracle map invertible-enough to search?  [partially-tractable]

**Verdict:** Solvable in principle, but only with the right structure — blind gradient-free search over an arbitrary medium against TRIBE at 600s/query is NOT practical. The hard wall is query budget x dimensionality: the near-exact published analog (Black-Box Adversarial Prompting, arXiv:2302.04237) uses TuRBO/Square-Attack with token-space projection and a smooth real-valued loss — exactly our scoreActivations cosine blend, which is the good news — yet still spends 5,000-10,000 black-box queries to tune only 4-6 tokens. CMA-ES is built for 3-100 dims and degrades fast above that; vanilla BO collapses past ~15-100 dims. So raw search over text-as-tokens or media-as-pixels is research-grade-to-infeasible at our oracle cost. What makes the oracle 'invertible-enough' to search is (a) a SMOOTH, real-valued score (we have it; binary scores are dead on arrival) and (b) drastically reduced effective dimensionality. The single best path is to NOT search raw outputs but to search the low-dimensional LATENT of a per-medium generative model and let derivative-free search (TuRBO preferred over CMA-ES; NES as fallback) move the latent to match the target — this is precisely NeuroGen / deep-generator activation maximization (arXiv:1605.09304, NeuroImage 2021), which already does Volta's task (synthesize an output that hits a TARGET neural-encoding activation) by optimizing a generator latent, no gradients-through-oracle and crucially NO reasoner-about-the-medium required: the generator carries the structural prior and guarantees renderability. Make it affordable by wrapping it with a surrogate of TRIBE (surrogate-assisted CMA-ES/TuRBO) so most evaluations are cheap, and gate with trust regions to resist the reward-hacking we already documented. For per-run efficiency, an amortized inverter (vec2text-style corrector, ~50 queries vs 5,000; ZSInvert for zero-shot) is the strongest option where a corrector can be trained on (output -> TRIBE activation) pairs. Volta's current genetic-operator loop is the correct honest fallback and the right OUTER wrapper, but as a standalone it is the least sample-efficient and degrades on rugged landscapes. Bottom line: partially-tractable — feasible per-medium when you supply a generative prior (or corrector) for that medium plus a surrogate to cut oracle calls; not feasible as a single generic blind search.

- **Direct gradient-free search over a continuous latent + token-space projection (CMA-ES / TuRBO / Square Attack, the 'Black-Box Adversarial Prompting' recipe)** (expensive): Run a derivative-free optimizer (CMA-ES, NES, or Trust-Region BO) over a low-dim continuous embedding vector, project each candidate to the nearest discrete tokens / render it, query TRIBE, and use neural-similarity as the real-valued fitness.
  - fit: This is the closest published analog to Volta: Maus et al. optimize 4-6 tokens by running TuRBO/Square-Attack in CLIP/BERT embedding space, projecting to tokens (Token-Space Projection), with ZERO gradient access and only a real-valued black-box loss. Our scoreActivations() cosine blend IS exactly such a smooth real-valued loss (they explicitly note binary losses give 'too sparse a signal'). For text it drops in: optimize a small embedding, project to tokens, render, score. For arbitrary media it generalizes ONLY if you have a renderer that turns a continuous vector into that medium.
  - requires: A LOW-dimensional continuous parameterization (they cap at 4-6 tokens => 3,072-dim; CMA-ES degrades badly above ~100 dims and is quadratic in dim; BO/GP collapses past ~15-100 dims absent trust regions/TuRBO). A smooth, real-valued (not binary) oracle map. A way to map continuous->renderable (token projection, or a generator). NO gradients needed.
- **Latent-space search with a generative prior (deep-generator activation maximization / NeuroGen)** (research-grade): Don't search raw outputs; search the LOW-dim latent code of a frozen generative model for the output medium (a text LM's prompt/latent, a Flux/GAN image latent), so every candidate the optimizer proposes already decodes to a realistic artifact, and only the latent is optimized to match the target activation.
  - fit: This is the single most important enabler for the 'arbitrary medium, no cross-modal reasoner' goal. Nguyen et al.'s deep-generator activation maximization and NeuroGen synthesize images that hit a TARGET neural-encoder activation by optimizing a generator's latent code, not pixels — directly analogous to matching a TRIBE target. The generator prior collapses a million-pixel/discrete-token space into a smooth ~100-1000-dim manifold of valid artifacts, which is what makes derivative-free search converge AND guarantees renderability. For Volta: per medium, plug in a generator (we already have Flux for images), then run CMA-ES/TuRBO/NES over its latent against scoreActivations. No reasoner about the medium is required — the generator carries the structural prior.
  - requires: A trained generative model whose latent decodes to valid artifacts IN THE TARGET MEDIUM (image: have it; text: an LM; novel media: must train/obtain one). The generator must be reasonably smooth so the latent->activation map has usable gradient signal under sampling. Still many oracle queries (hundreds-thousands), so a fast/cheap oracle or surrogate is needed to be practical.
- **Learn-to-invert: train an amortized corrector/inversion model (vec2text / ZSInvert)** (expensive): Instead of searching from scratch every run, train a model that maps a target activation vector back to a candidate output, then iteratively re-query the oracle and correct toward the target — amortizing the search across many runs.
  - fit: vec2text is essentially Volta-for-text-embeddings already: it inverts a frozen black-box embedder by guessing text, re-embedding, and correcting to minimize cosine distance — gradient-free, query-only. It hits 92% exact recovery of 32-token text in ~50 queries (vs 5,000 for blind BO above) BECAUSE the corrector front-loads the search. For Volta this is the most query-efficient path PER RUN, and per-medium it just needs a corrector trained on (output -> TRIBE activation) pairs. ZSInvert removes the per-oracle retraining (zero-shot, any embedder). The catch: our oracle output is a [T,20484] trajectory, not one sentence-embedding, so the corrector must be (re)trained for TRIBE's output shape and each output medium.
  - requires: A dataset of (rendered output -> TRIBE activation) pairs to train the corrector (vec2text used ~5M pairs PER embedder); or a zero-shot inverter (ZSInvert) if one generalizes to TRIBE. An iterative re-query loop. NO gradients. Heavy upfront/amortized cost, cheap per-run.
- **Surrogate-assisted optimization: fit a cheap proxy of TRIBE to slash real queries** (medium): Train a fast neural/GP surrogate that predicts the TRIBE activation (or directly the similarity score) from a rendered output, optimize aggressively against the cheap surrogate, and spend the scarce real-TRIBE queries only to validate/refit the surrogate online.
  - fit: Directly attacks our binding constraint — the 600s/query oracle. Surrogate-assisted CMA-ES and BO are the standard fix for 'expensive, noisy, feasibility-limited' objectives. For Volta, every real-oracle approach above (latent CMA-ES, BO, inversion) becomes affordable if most candidate evaluations hit a learned surrogate of TRIBE instead. It is medium-agnostic at the SCORE level: the surrogate maps a rendered artifact's features to a predicted activation/score, no reasoner needed. Risk: the optimizer reward-hacks surrogate error (we already documented TRIBE-score reward hacking), so it needs trust-region/uncertainty gating and periodic real-oracle checks.
  - requires: An online training loop and a representation of rendered artifacts the surrogate can consume; budget to seed the surrogate with some real TRIBE queries; uncertainty handling to avoid exploiting surrogate error. Pairs naturally with TuRBO (local GP surrogate) or latent CMA-ES.
- **Population metaheuristics directly on the structured object: GA / simulated annealing / particle swarm over tokens or media operators** (cheap): Skip continuous relaxation entirely: maintain a population of full outputs, apply medium-specific mutation/crossover operators (Volta's evolutionary operators), accept/reject by TRIBE-score with SA-style temperature, and let selection climb the landscape.
  - fit: This is literally Volta's CURRENT loop (genetic operators: elite, point mutation, crossover, ablation, novelty, reset over output nodes) — the honest, medium-agnostic baseline that needs neither gradients, a generator prior, nor a reasoner, only a mutation operator per medium and the score. It scales to ARBITRARY structured media as long as you can define one valid mutation. The catch is sample efficiency and landscape ruggedness: SA/GA need many evaluations and degrade sharply as the fitness landscape gets rugged (NK-landscape studies show GA underperforms SA as ruggedness/size grow), and our 600s/query budget caps the population/generations hard. Best used as the universal fallback and as the OUTER loop wrapping the cheaper inner methods above.
  - requires: Only a mutation/crossover operator that keeps outputs valid in the medium, and the real-valued score. NO continuous structure, NO generator, NO reasoner. But assumes the landscape is climbable (not needle-in-haystack flat) and that you can afford many serial oracle queries.

### learned-inverse: Can we train a decoder g(activation)→x or a conditional generator so that f(g(a))≈a for the frozen TRIBE encoder, per output medium? What data/queries does it need, and is it feasible for an arbitrary medium queried slowly with no cross-modal reasoner?  [partially-tractable]

**Verdict:** A learned inverse is partially-tractable for Volta — but it is NOT the generic answer, and the literature actually argues against needing one. The decisive evidence: the canonical brain decoder, Tang et al. 2023 (Nature Neuroscience), does NOT train a brain→text inverse. It keeps the frozen ENCODING model and runs a beam search where a language model proposes word continuations and the encoder SCORES them by predicted-vs-actual response likelihood (beam width 200, up to 5 continuations/candidate, encoder fit on 16h of paired fMRI). That is structurally identical to Volta's current generator→render→score→judge loop — so the single best path is to keep TRIBE as a frozen SCORER and invest in better proposers, not to invert TRIBE. Where a true inverse pays off, it is as an offline WARM-START proposer, trainable WITHOUT gradients and WITHOUT a cross-modal reasoner via the inverse-design / amortized-SBI recipe: sample many renderable outputs x for one medium, query TRIBE to build (x, f(x)) pairs, fit g(a)→x (tandem-net / cGAN / neural-posterior to handle one-to-many non-uniqueness). Hard constraints that make this only PARTIALLY tractable: (1) it is per-medium, not zero-shot — it needs a samplable renderer/parameterization for the medium, and Volta's renderers are still MVP type-signatures (docs/IO_MODULES.md Scaffold Status), so there is nothing to sample yet for most media; (2) it costs a large upfront query budget (SBI/inverse-design literature: ~10k–100k forward evals), and Volta's hosted TRIBE is deliberately slow/low-concurrency (VOLTA_ORACLE_TIMEOUT_MS=600000, VOLTA_SCORING_CONCURRENCY=1), so the budget is the binding constraint; (3) inversion is fundamentally ill-posed (many-to-one) and degenerates into adversarial 'fooling' outputs unless regularized by a generative prior over the medium — i.e. you cannot escape needing a generative model of the medium, you only move where it lives. The MindEye2/unCLIP shortcut (map activation→CLIP→off-the-shelf generator) is realistic ONLY for image output (activation→CLIP→Flux), because it borrows another medium's already-solved inverse and a frozen shared embedding; it fails by construction for an arbitrary medium with no foundation embedding and no ready generator — precisely the case the brief flags. Net: for the genuinely-arbitrary-medium requirement, don't bet on a learned inverse as the primitive — bet on the frozen-encoder-as-scorer + better proposer (Tang pattern), and use a learned per-medium inverse only opportunistically as a warm-start once a samplable renderer for that medium exists and the query budget is affordable. - **Encoding-model + generative search (NO learned inverse) — the Tang et al. brain-decoder pattern** (cheap): Don't invert at all: keep TRIBE as the frozen scorer and let a domain generator propose candidates that the oracle ranks by predicted-vs-target activation likelihood — exactly Volta's current loop.
  - fit: This IS Volta's loop, and it is the literature's actual answer to 'decode a frozen brain-encoding model.' For an arbitrary medium you only need (a) any candidate proposer over that medium's renderable states (LLM today, but could be a random/grammar/parametric sampler) and (b) the existing scoreActivations() in packages/core/src/scoring/activation.ts. No inverse, no gradients, no cross-modal reasoner strictly required — a blind proposer + oracle ranking still works, just less sample-efficiently. The cross-modal reasoner only makes the proposal distribution smarter.
  - requires: A frozen encoder used as a scorer (have it) and any proposer over the medium. Scales with oracle query budget; a dumb proposer needs many more queries. Assumes you can render+score candidates, which Volta already does via the renderer→TRIBE path in tribe_oracle_worker.py.
- **Inverse-design net / tandem network / cGAN trained on self-generated (x, f(x)) pairs** (expensive): Sample many renderable outputs x for a medium, query TRIBE to get a=f(x), then train g(a)→x (or a conditional generator g(a)→x) offline so that at inference one forward pass proposes an x whose activation matches a target a*.
  - fit: Concretely: pick a medium (e.g. text or Flux image latents), draw 10k–100k samples x from a renderer/parameterization, batch them through the hosted TRIBE oracle to build a paired dataset, then fit a per-medium inverse. At run time g(target_activation) emits a warm-start candidate that the existing loop refines. Works for an ARBITRARY medium IFF you have a way to SAMPLE x for that medium (a renderer + a sampling distribution over its inputs) — no gradients through TRIBE and no cross-modal reasoner needed.
  - requires: A samplable parameterization/renderer for the medium (Volta has renderers as MVP type-sigs only — this is the gating dependency). A large upfront query budget (inverse-design and SBI papers use ~10k–100k forward evals). Inherits the one-to-many non-uniqueness problem: many x map to similar a, so a plain regression inverse collapses; needs tandem (forward-fixed) or cGAN/MDN formulation. Per-medium retraining; not zero-shot to a new medium.
- **Amortized / simulation-based inference (SBI) posterior over outputs** (expensive): Treat TRIBE as a black-box simulator a=f(x) and train an amortized network q(x|a) (neural posterior estimation) on simulated (x,a) pairs; at test time it returns candidate outputs for any target a in one pass.
  - fit: Same data-generation recipe as the inverse-design approach (query TRIBE on sampled x), but the formal SBI framing gives a DISTRIBUTION over matching outputs, which directly addresses non-uniqueness (sample several x, score each with the real oracle, keep the best). Useful if Volta wants a fast proposer that returns a diverse candidate set to feed the score-judge loop rather than one point estimate.
  - requires: A prior/sampler over the medium's outputs and a sizable simulation budget (SBI practical guides cite 10k–100k sims; active-learning proposals cut this but tie it to one target, losing amortization). The oracle query must be cheap enough to run thousands of times — Volta's hosted TRIBE is slow (VOLTA_ORACLE_TIMEOUT_MS default 600000), so the budget is the real constraint. No gradients needed.
- **Shared-embedding decoder + off-the-shelf generator (the MindEye2 / unCLIP recipe)** (medium): Learn a small mapping from TRIBE activation into a pre-aligned embedding space (e.g. CLIP), then hand that embedding to an EXISTING generative model (SDXL unCLIP) that already inverts the embedding into the medium.
  - fit: Only works where the medium has (a) a foundation embedding space TRIBE activations can be linearly/MLP-mapped onto and (b) a ready generator that inverts that space. MindEye2 needed only ~1 hour of paired (brain, image) data precisely because OpenCLIP stayed frozen and SDXL unCLIP did the heavy lifting. For Volta this is realistic for IMAGE output (TRIBE-activation→CLIP→Flux/SDXL) but NOT for an arbitrary medium with no foundation embedding and no off-the-shelf generator — exactly the case the SYSTEM prompt flags ('no model can reason about that medium').
  - requires: A frozen shared embedding for the medium AND an off-the-shelf generator that inverts it, plus paired (activation, embedding) data to fit the mapper. This is the LEAST generic option: it borrows another medium's already-solved inverse. Fails by construction for novel/arbitrary media.
- **Direct model inversion / activation maximization over renderable outputs** (research-grade): Optimize an output x to make f(x) match target a* directly (gradient or gradient-free), i.e. invert the network per-target without training a reusable decoder.
  - fit: Per-target, no training set, no cross-modal reasoner. But TRIBE is a black box with no exposed gradients, so you are stuck with gradient-free search (CMA-ES / BO) over the medium's parameters — which is just Volta's existing loop with a different proposer, and high-dim BO degrades past ~15 dims. Worse, code-inversion/activation-maximization is well-documented to produce non-realistic 'fooling' outputs unless a learned natural-output prior regularizes it — reintroducing the need for a generative model of the medium.
  - requires: Either gradients through f (we DON'T have them — black-box oracle) or a gradient-free optimizer plus a strong natural-output prior to avoid adversarial/fooling solutions. Many slow oracle queries per single target, no amortization across targets. Ill-posed (many-to-one) without the prior.

### embedding-search: Can we search the latent space of a pretrained generative model of the target medium (GAN/VAE/diffusion) to find an output whose frozen black-box TRIBE activation matches a captured target signature — when the medium may have NO cross-modal reasoner — using latent optimization / CLIP-guided generation / score-based guidance / projected gradient via surrogate?  [partially-tractable]

**Verdict:** Solvable in principle, with one dominant practical path — but only for media that HAVE a pretrained generator, and the win is bounded by Volta's slow oracle. The honest landscape: (1) the popular 'embedding-search' methods you'd reach for first — CLIP-guided generation, FuseDream, ReNO, score-based diffusion guidance — all assume a DIFFERENTIABLE scorer and backprop through it. TRIBE exposes no gradients (confirmed in tribe_oracle_worker.py: stdin/stdout JSON returning a values tensor, zero autograd surface), so these DO NOT transfer unmodified and are a trap. (2) The methods that genuinely fit Volta's frozen-black-box-queried-slowly constraint are the derivative-free ones: a gradient-free latent/noise search (CMA-ES or ZeNO) is the single best fit — it needs nothing but the scalar TRIBE-similarity Volta already computes, is proven on GANs and one-step diffusion with NON-differentiable rewards, and is medium-agnostic (swap the generator, keep the loop). Wrap it in Bayesian optimization (Tripp et al.'s weighted retraining hit useful results in ~500 evaluations) to fight the slow oracle, and keep the latent low-dimensional or trust-region-restricted or it won't converge. The two hard caveats: (a) it inherits Volta's already-noted reward-hacking risk — pure score-maximization in latent space drives outputs off the natural manifold and games the metric, so on-manifold regularization / projected updates / a held-out check are mandatory, not optional; (b) it presupposes a generator for the medium, so it does NOT by itself solve a medium with no generator. If you can afford a big offline query budget, the higher-ceiling but research-grade plays are a differentiable surrogate of TRIBE (recovers gradients, amortizes across targets, mirrors fMRI-to-image decoders) or a fully amortized signature->latent inverse (one forward pass per target at run time, like production brain-decoding pipelines). RECOMMENDATION for Volta now: prototype CMA-ES over a pretrained generator's latent driven by the existing TRIBE-similarity score, keep candidates on-manifold, and budget for hundreds of oracle calls per target. Sources: Tripp et al. 'Sample-Efficient Optimization in the Latent Space of DGMs via Weighted Retraining' (NeurIPS 2020, arXiv:2006.09191); 'Gradient-Free Noise Optimization / ZeNO' (arXiv:2605.11347); FuseDream (arXiv:2112.01573); ReNO / 'Controlling Latent Diffusion Using Latent CLIP' (arXiv:2503.08455); 'Mitigating over-exploration in LSO using LES' (arXiv:2406.09657); 'Robustify Black-Box ML / Zeroth-Order' (arXiv:2203.14195); Takagi and Nishimoto 'High-resolution image reconstruction with LDMs from human brain activity' (bioRxiv 2022.11.18.517004); reward-hacking evidence in 'Projected Gradient Ascent for Reward-Guided One-Step Generative Models' (arXiv:2602.08646).

- **Gradient-free latent / noise optimization with an evolution strategy (CMA-ES, or ZeNO-style particle/path-integral noise search)** (medium): Treat the generator latent (GAN z, VAE z, or diffusion init-noise) as the decision variable and run a population-based, derivative-free optimizer that only reads scalar TRIBE-similarity values to drive z toward an output whose activation matches the target.
  - fit: This is the single best literal fit: it needs nothing from TRIBE except a scalar score per candidate, exactly what tribe_oracle_worker.py already returns (cosine/L2 of the [T,20484] values vs the captured target). For ANY medium with a pretrained generator, you optimize z in that generator's native latent space and render z -> artifact -> TRIBE -> score. ZeNO explicitly handles NON-differentiable black-box rewards (e.g. GenEval object-detection score) on GANs and one-step diffusion (SDXL-Turbo, DMD2, DFGAN, LCM), which is precisely the 'no gradients exposed' regime. CMA-ES is the workhorse version (cma pip package). No cross-modal reasoner is required: the generator supplies the manifold, TRIBE supplies the signal — the search itself does the 'reasoning'.
  - requires: A pretrained generator for the medium (the hard prerequisite). A reasonably low-dimensional, well-behaved latent (CMA-ES is happy in ~tens-to-few-hundred dims; struggles past ~1000 without restriction to a subspace). MANY oracle queries: ZeNO uses N=16 particles x M iterations, CMA-ES needs population_size ~ 4+3*ln(d) per generation x dozens-to-hundreds of generations => easily hundreds to thousands of TRIBE calls per target. Assumes the score landscape is exploitable by sampling, not that it is smooth or differentiable.
- **Bayesian optimization in latent space (GP/TuRBE/LES + weighted retraining)** (medium): Build a surrogate (Gaussian process or similar) over the generator's latent space that predicts TRIBE-similarity, pick the next latent to evaluate via an acquisition function (expected improvement), and only spend a real TRIBE query where the surrogate is most promising.
  - fit: Directly targets Volta's worst constraint — the oracle is SLOW — by being the most query-efficient black-box method available. Tripp et al. (NeurIPS 2020) hit strong results in only ~500 objective evaluations, and the latent-space framing is exactly 'optimize structured/discrete outputs via a generative model's continuous latent.' Their 'weighted retraining' trick (re-train the generator to concentrate latent mass on high-TRIBE regions) and LES (over-exploration guard) keep candidates on-manifold so you don't waste queries on garbage outputs. Medium-agnostic: swap the generator, keep the BO loop.
  - requires: Same pretrained generator. Crucially, GP-BO scales badly with dimension — needs the latent kept low-dim (<~20-60 raw, or a learned/trust-region subspace like TuRBO) or it collapses. Weighted retraining additionally assumes a LARGE labelled dataset to (re)train the generative model, and that the objective is deterministic (TRIBE is — confirmed). Still needs an initial pool of queries to seed the GP.
- **Surrogate-of-the-oracle: learn a differentiable proxy of TRIBE, then backprop through generator+proxy (projected gradient via surrogate)** (expensive): Train a small differentiable network f_hat(artifact_features) ~ TRIBE-activation by querying the real TRIBE on a corpus of rendered outputs, then do gradient ascent on the generator latent through render -> f_hat to match the target, periodically re-querying real TRIBE to correct the surrogate.
  - fit: Recovers the gradient that TRIBE refuses to expose, enabling fast inner-loop latent optimization and amortizing oracle cost across many targets (one surrogate serves all future runs in that medium). This is the standard 'differentiable surrogate for a black-box / non-differentiable physical layer' pattern, and is essentially what the fMRI->image reconstruction literature does (train a decoder from many (brain-activation, image) pairs, e.g. Takagi & Nishimoto LDM-from-fMRI), repurposed as a forward surrogate. For a medium with no reasoner, the surrogate IS the learned reasoner.
  - requires: A substantial offline query budget to build the (output -> TRIBE) training set, and a differentiable render path from latent to TRIBE-input features (image/video renders are differentiable; text/code tokenization is NOT, which breaks this route for discrete media unless you stay in a continuous generator latent). Surrogate fidelity is the risk: gradients of f_hat off-distribution are unreliable, so it needs trust-region steps + real-TRIBE recalibration. Assumes the oracle map is smooth enough to approximate.
- **CLIP-guided / score-based guidance with TRIBE in the conditioning slot (ReNO / FuseDream-style)** (research-grade): Use the generator's own guidance machinery — diffusion classifier/score guidance or GAN+CLIP latent optimization — but plug TRIBE-similarity in as the reward/guidance signal instead of a text-CLIP score.
  - fit: Appealing on paper because it's the dominant paradigm (ReNO noise optimization, FuseDream CLIP+GAN), BUT these methods assume a DIFFERENTIABLE scorer: FuseDream backprops through CLIP, ReNO/ORIGEN/score-distillation take gradients of the reward. TRIBE exposes no gradients, so the textbook versions DO NOT apply unmodified. They only become usable for Volta after pairing with the surrogate above (to supply gradients) or by degrading to their gradient-free fallbacks (Best-of-N / ZeNO), at which point they collapse into approach #1. Listed mainly to flag the trap.
  - requires: A DIFFERENTIABLE oracle (Volta does NOT have one) or a trained differentiable surrogate. For diffusion-guidance variants, also a diffusion generator specifically. Inherits the whole reward-hacking failure mode.
- **Amortized inversion: train a feed-forward map target-TRIBE-signature -> generator latent** (research-grade): Offline, sample latents z, render, query TRIBE to get activations a; train an inverse network g(a) -> z so that at run time a single forward pass turns any target activation into a matching latent — no per-target search.
  - fit: This is the 'do the expensive thing once, then it's free per target' play and is exactly how production fMRI-to-image decoders work (amortized, one forward pass, no per-image oracle loop). If achievable it removes Volta's slow-oracle pain entirely at inference and is fully medium-agnostic (retrain g per medium). Pairs naturally with light test-time latent refinement (a few CMA-ES steps) to fix residual mismatch.
  - requires: A LARGE offline (latent, TRIBE-activation) dataset per medium and a pretrained generator. The inverse is genuinely hard — one network inverting the whole generator+TRIBE composition (the known weakness of amortized inference; mitigated by hypernet/instance-adaptive tricks). The mapping is one-to-many (many outputs share a vibe) so it learns an average; needs refinement to sharpen. Heavy up-front investment, not a hackathon-weekend build.

### medium-agnostic-framework — Can a shared representation (specifically a TRIBE [T,20484] cortical-activation vector) act as the conditioning vector for a generic conditional generator of ANY output medium, and what is the minimal model stack to add a new output medium generically, given a frozen black-box oracle queried slowly with no gradients?  [partially-tractable]

**Verdict:** Yes in principle, but NOT as a free 'plug TRIBE in as a universal conditioning vector' move. The clean dream — feed a 20484-dim TRIBE activation straight into a generic conditional generator of any medium — fails on two hard, confirmed constraints: (a) TRIBE-brain-space is alien to every off-the-shelf generator's conditioning space, and (b) we have no gradients and a non-differentiable render boundary (verified in renderers/* and scoring/activation.ts), so every inference-time-guidance / shared-aligner method (ImageBind aligners, Seeing-and-Hearing) is categorically ruled out, and any-to-any models (CoDi, Unified-IO 2) condition on their OWN latent, not ours. What DOES generalize is the inverse-problem framing: TRIBE is a brain ENCODER, so 'render an output that matches activation A' is exactly fMRI-to-stimulus decoding, and the literature (ID3PM black-box inversion 2303.13006, RCG 2312.03701, MindEye/CnD brain-decoders) gives a real, medium-agnostic recipe — manufacture a (output, activation) dataset by self-querying the frozen oracle, then train an oracle-conditioned generator per medium. SINGLE BEST PATH, two-tier: (1) NOW / generic / zero training — black-box latent-space optimization (CMA-ES/Bayesian opt over a frozen per-medium generator's surrogate latent, per arXiv 2509.23800), which is just Volta's existing search loop with a continuous knob, needs only slow forward queries, no gradients, no paired data, and works for ANY medium that has any generator; (2) to scale past slow per-target search, amortize it into a trained activation->medium decoder (ID3PM-style CFG diffusion) whose training data you generate by querying TRIBE — expensive once per medium, then inference is a single fast forward pass with the oracle out of the loop. MINIMAL STACK to add a new medium generically = {a frozen generator (or sample corpus) for that medium} + {the render+TRIBE-query wrapper as a scalar fitness} + {a black-box optimizer}; upgrade to {+ a trained activation->latent adapter/decoder} only when query cost forces amortization. Sources: CoDi arXiv 2305.11846; ImageBind arXiv 2305.05665; Unified-IO 2 arXiv 2312.17172; RCG arXiv 2312.03701; ID3PM (black-box inversion) arXiv 2303.13006; CnD brain decoding arXiv 2310.00318; Seeing-and-Hearing aligners arXiv 2402.17723; surrogate-latent black-box optimization arXiv 2509.23800; CLIP-as-prompt-converter arXiv 2305.12716; unCLIP/DALLE-2 arXiv 2204.06125.

- **Black-box latent-space optimization over a frozen per-medium generator (search, no training)** (medium): Keep the existing search loop but make it generic: take any off-the-shelf generator for the new medium, optimize a low-dim latent (CMA-ES / Bayesian opt) so its rendered output's TRIBE activation matches the target — exactly Volta's loop with a continuous knob instead of an LLM proposer.
  - fit: This is the only approach that works TODAY for an arbitrary medium with NO cross-modal reasoner and NO training. 'Defining latent spaces by example' (arXiv 2509.23800) builds non-parametric surrogate latent spaces from ANY frozen generator (diffusion/flow, images/audio/video/proteins) with no extra training, and optimizes a black-box, gradient-free, expensive objective over them — our TRIBE-similarity score is exactly that objective. To add medium X: drop in a deterministic generator for X (DDIM/flow), wrap render+TRIBE-query as the scalar fitness, run CMA-ES/BO over the surrogate latent. No LLM, no gradients, no paired data needed. The render boundary being non-differentiable (confirmed: text->events, image->mp4, code->screenshot->video, custom multi-view cosine in scoring/activation.ts) is a non-issue because the optimizer never differentiates.
  - requires: A frozen, deterministic generator for the new medium (DDIM/flow with a latent that fully specifies output). Many slow oracle queries (BBO needs dozens-to-hundreds of evals; Table 2 in 2509.23800 uses 200 CMA-ES iters; ID-style problems converge in dozens in low-dim surrogate spaces). Objective assumed only evaluable, not smooth or differentiable. Scales well only to ~3-100 effective dims, so you optimize a compressed/surrogate latent, not raw pixels.
- **Amortized decoder / black-box inversion: train an oracle-conditioned generator per medium** (expensive): Treat 'render an output matching activation A' as decoding a stimulus from a brain activation — TRIBE IS a brain encoder, so this is literally fMRI-to-stimulus decoding. Train a conditional diffusion/AR generator that maps a TRIBE activation -> an output in medium X, using a dataset of (output, activation) pairs you MANUFACTURE by self-querying the frozen oracle.
  - fit: This is THE genuinely medium-agnostic recipe and the right long-term answer. ID3PM (Disney, arXiv 2303.13006) inverts a frozen BLACK-BOX face recognizer with only forward embeddings — no gradients — by training a classifier-free-guidance diffusion model on (image, embedding) pairs (69k FFHQ images), then at inference it is FULLY AMORTIZED: zero oracle queries to decode a new target. Brain-decoding (MindEye, Takagi-Nishimoto, CnD arXiv 2310.00318) does the same fMRI-activation->image. RCG (arXiv 2312.03701) shows a frozen representation can drive a pixel generator. The recipe per new medium X: (1) generate a corpus of medium-X samples, (2) render+query frozen TRIBE to get each one's activation, (3) train a small conditional generator activation->X with CFG. Inference is then a single fast forward pass, no slow oracle in the loop. The non-differentiable render boundary doesn't matter — you only ever query forward.
  - requires: A way to sample MANY plausible outputs in medium X (a base generator or a corpus), plus a large up-front budget of slow oracle queries to label them (thousands of forward passes — the one-time training cost). Per-medium training run. Assumes the activation carries enough signal to be invertible at all (a real risk: TRIBE pools to brain-region responses, may be many-to-one). Does NOT need gradients of TRIBE, does NOT need a cross-modal reasoner.
- **Trained adapter from TRIBE-space into an existing generator's native conditioning space** (medium): Train one lightweight MLP/projection that maps a TRIBE [T,20484] activation into the conditioning space a powerful frozen generator already understands (e.g. CLIP / T5 embeddings for image/text/video diffusion), then generate with that off-the-shelf model untouched.
  - fit: Cheaper bridge than full inversion and reuses SOTA generators. Multiple works show a small MLP can project an alien embedding into CLIP space to drive Stable-Diffusion-class models without retraining the generator ('CLIP is secretly an image-to-prompt converter' 2305.12716; Face0; unCLIP/DALLE-2 2204.06125 maps CLIP-image->image). ImageBind even reuses a DALLE-2 decoder for audio->image because audio was bound INTO CLIP space. For Volta: add medium X by picking a frozen X-generator with a known conditioning space, train adapter (TRIBE activation -> that space) on (activation, target-embedding) pairs harvested by self-querying. Only works for media that ALREADY have a strong conditioning-space generator (image/video/text); for a truly novel medium with no such generator you fall back to approach 1 or 2.
  - requires: A target generator whose conditioning space is documented/usable, and paired data to fit the adapter (again harvested by querying TRIBE on a labeled corpus). Assumes a learnable, roughly-aligned map exists between TRIBE-brain-space and the generator's space — plausible for semantic content, weak for the affective 'vibe' TRIBE rewards (scoring.ts notes TRIBE rewards emotional response, not description), so adapter may transfer topic but lose vibe.
- **Off-the-shelf any-to-any model as the generator backbone (CoDi / Unified-IO 2)** (medium): Use a pretrained any-to-any multimodal model (CoDi, Unified-IO 2) as the generator and let TRIBE be only the scorer/selector — i.e. these supply the 'generic generator across media,' Volta supplies the brain-matching objective.
  - fit: Useful as the candidate PROPOSER for media these models cover (text/image/audio/video/action), replacing the LLM. CoDi (arXiv 2305.11846, NeurIPS'23) builds a shared latent and generates any combination of modalities; Unified-IO 2 (2312.17172) tokenizes image/text/audio/action into one space. But they condition on THEIR OWN shared latent, NOT on TRIBE-brain-space — there is no way to feed a 20484-dim cortical vector in as the condition. So they cannot be 'conditioned on the shared representation' directly; they only enlarge the set of media you can sample from inside approach 1 (search) or supply training samples for approach 2/3. They also cover only their trained modalities, so they do not solve the truly-arbitrary-medium case.
  - requires: The new medium must be one the any-to-any model already supports. No bridge from TRIBE-space into their condition exists, so still need search (approach 1) or a trained map (approach 2/3) on top. Heavy models, but pretrained/frozen.
- **Inference-time gradient guidance through a differentiable binder (ImageBind aligners) — RULED OUT** (research-grade): Guide a frozen generator's latent at inference by backpropagating an alignment score through a differentiable shared-embedding binder, with no per-medium training.
  - fit: This is the seductive 'free lunch' and it does NOT apply to us. 'Seeing and Hearing' (arXiv 2402.17723) optimizes generator latents by gradient-ascent on ImageBind alignment, with gradients flowing through the frozen binder; the authors' method 'inherently depends on backpropagating alignment scores.' Our oracle is BLACK-BOX with no exposed gradients, and even if TRIBE's PyTorch weights were reachable, the render boundary (text->tokenized events, image->mp4 via moviepy, code->screenshot) is non-differentiable, severing any gradient path to the score. Listed explicitly so the team does not chase it: any method needing gradients through the binder is categorically unavailable in Volta's setup.
  - requires: Differentiable, white-box binder AND differentiable render path — neither of which Volta has. Included as a negative result.

  - fit: This is the AutoML/meta-BO angle: treat each run as one evaluation of a configuration and meta-learn the configurator. Concretely for Volta: log (targetBucket, chosen candidateCount/maxIterations/diversity/explorationWeight, finalBestNeuralSimilarity) per run; fit a tiny model (even a lookup table or multiple-adaptive-Bayesian-linear-regression / deep-kernel surrogate from the transfer-BO survey) that initializes those knobs and planOperators priors for a new target by nearest target type. Solves the documented cold-start problem in transfer-BO. Medium-agnostic because the descriptor is the activation/medium and the knobs are generic. BUT: this needs DOZENS-to-HUNDREDS of past runs to be worth more than the bandit warm-start in approach 1, and each Volta run is expensive (slow oracle, M*N queries), so the meta-dataset accrues slowly. For a hackathon-scale corpus it is over-engineered relative to the bandit table.
  - requires: A sizable corpus of completed runs with comparable scores; a meaningful, stable target-descriptor space; a surrogate model. Assumes smoothness/transfer of the run->outcome map across target types. Heaviest data requirement of all options; the slow oracle makes the meta-dataset the bottleneck.
- **AlphaEvolve-style self-rewriting of the operator library itself (full self-improvement)** (research-grade): Let the controller not just SELECT among fixed operators but EVOLVE the operator set: an LLM proposes new operator instructions, retires consistently-losing ones, and tunes the planner code/prompts, with the run-outcome score as the meta-fitness — PromptBreeder's self-referential layer applied to Volta's strategy library.
  - fit: Most ambitious and the truest 'self-improve from its own experiments.' PromptBreeder shows an LLM can mutate the mutation-operators themselves; AlphaEvolve shows a controller can edit its own code under an evaluator. For Volta: periodically have a meta-LLM read operatorFitnessStats across many targets and propose edits to refinementStrategies/coldStartStrategies (add a new register, reword a weak one, delete a dead one), validated by whether the edited library raises best neuralSimilarity on a held-out set of past targets. This is medium-agnostic in principle because operators are written as medium-neutral 'registers/behaviors' (the code comments already insist on this). HONEST CAVEAT: this is research-grade for our constraints — it needs a held-out target benchmark to score library edits, many oracle queries per meta-iteration, and it amplifies reward-hacking risk (the meta-loop will happily evolve operators that game cosine). Highest payoff, lowest near-term tractability.
  - requires: A trustworthy held-out evaluation set of targets, a hardened (non-gameable) score metric FIRST, an LLM meta-controller, and a large oracle-query budget for meta-fitness. Assumes the score is not exploitable — currently false per the reward-hacking memo. No gradients, but very many oracle queries.
